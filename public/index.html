VisionGen — Deploy Guide

// From your files to live on the internet in 15 minutes
// First — Your Folder Structure
How to organize your files
Create a folder on your computer called visiongen and set it up exactly like this:
visiongen/ ├── server.js ← the backend file I gave you ├── package.json ← the config file I gave you └── public/ └── index.html ← your VisionGen HTML app (rename it)
The public/ folder is where your VisionGen HTML file lives. Rename it to index.html.
// Step 01
Get your WaveSpeed API Key FREE CREDITS
Go to wavespeed.ai → Sign up free → Go to API Keys → Create a new key.

It looks like: wsa_xxxxxxxxxxxxxxxxxxxx

Copy it — you'll need it in Step 3.
// Step 02
Create a GitHub account & upload your files
Render (the free hosting platform) deploys directly from GitHub.

1. Go to github.com → Sign up free
2. Click New Repository → name it visiongen → Public → Create
3. Upload all 3 files: server.js, package.json, and your public/index.html

GitHub is just a place to store your code online. Render reads it from there.
// Step 03
Deploy on Render.com FREE
1. Go to render.com → Sign up with GitHub
2. Click New + → Web Service
3. Connect your visiongen GitHub repo
4. Fill in these settings:
Name: visiongen Environment: Node Build Command: npm install Start Command: node server.js Plan: Free
5. Scroll down to Environment Variables → click Add Variable:
WAVESPEED_API_KEY = wsa_your_actual_key_here
6. Click Create Web Service. Render builds and deploys automatically.
// Step 04
Get your live URL
After ~2 minutes Render gives you a URL like:

https://visiongen.onrender.com

Open it in your browser. Your full VisionGen app is live. Real AI video generation works automatically — no API key needed from the user because it's hidden on the server.
// Step 05 — Important
Remove the API key input from the frontend DO THIS
Since your API key now lives on the server, users don't need to enter their own key anymore.

In your index.html, find the WaveSpeed connect section and either:
— Remove it completely, or
— Change the label to say "Connected" by default

This keeps your key safe and gives users a cleaner experience.
// Verify It Works
Test your server is alive
After deploying, visit this URL in your browser:
https://your-app-name.onrender.com/health
You should see:
{ "status": "ok", "app": "VisionGen", "key": "configured" }
If you see "key": "configured" — you're good. If you see "missing" — go back to Render and check your environment variable.
// Free Plan Note
Render free tier goes to sleep
On the free plan, Render spins down your server after 15 minutes of no traffic. The first request after sleep takes ~30 seconds to wake up.

To fix this: upgrade to Render's Starter plan ($7/month) when you start getting real users. For now, free is fine.
